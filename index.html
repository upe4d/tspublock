Если у вас есть MTProxy сервер — запустите скрипт. Он 30 секунд собирает RST пакеты и отправляет новые RU IP в нашу базу.

Требования: Linux, root, iptables, curl. Запускать на сервере с MTProxy.

curl -fsSL https://stats.gptru.pro:4443/rst/rst_submit.sh | sudo bash -s -- ВАШ_ПОРТ 30

Замените ВАШ_ПОРТ на порт вашего прокси (обычно 443).

Дополнительные соображения

Механизм блокировки RST обсуждается в сообществе. Несколько версий:

Версия 1: ТСПУ шлёт RST со своих реальных IP — блокировка по IP работает.

Версия 2: ТСПУ подменяет IP клиента — тогда мы блокируем случайные адреса. Эффективнее блокировать по TTL пакета.

Версия 3: ТСПУ режет на стороне клиента — тогда блокировка RST на сервере вообще не влияет на handshake timeout.

Что знаем точно: ~8800 handshake timeout в час. Влияет ли блокировка — стата собирается.

CyberOK — задокументированные сканеры, блокировать полезно в любом случае. Автосбор — эксперимент. Обсуждение: @u_pre

Что происходит?

ТСПУ — «чёрные ящики» РКН на сетях операторов. При обнаружении MTProxy-подключения они вклиниваются в соединение и отправляют TCP RST пакеты со своих реальных IP на прокси-сервер.

Сервер получает RST и думает что клиент сам разорвал соединение — Telegram handshake timeout. Клиент видит что прокси не работает.

Решение — дропать RST пакеты от известных ТСПУ адресов через iptables + ipset.

CyberOK Skipa проверено раз в сутки

Задокументированный список IP адресов сканеров ТСПУ от независимых исследователей. Включает сканеры CyberOK, ГРЧЦ, НКЦКИ. Обновляется автоматически раз в сутки с GitHub.

github.com/tread-lightly/CyberOK_Skipa_ips

Автосбор RST эксперимент раз в сутки

Собираем IP которые шлют RST пакеты на порт 443. Фильтр: только российские адреса (RU по whois).

Важно: ТСПУ может подменять IP клиента при атаке — в таком случае мы собираем не ТСПУ, а провайдерское оборудование. Метод экспериментальный, точность уточняется сообществом.

Установка на свой сервер

1
Установить ipset
apt-get install -y ipset
2
Применить блок-лист CyberOK
Скачивает проверенный список и настраивает iptables:
curl -fsSL "https://stats.gptru.pro:4443/rst/api.php?action=export&fmt=iptables&src=cyberok" -o /tmp/ts.sh && bash /tmp/ts.sh
3
Проверить — pkts должны расти
iptables -L TSPUBLOCK -v -n
4
Сохранить правила
ipset save > /etc/ipset.conf && iptables-save > /etc/iptables/rules.v4
5
Автообновление CyberOK раз в сутки (3:00)
(crontab -l 2>/dev/null; echo "0 3 * * * curl -fsSL https://stats.gptru.pro:4443/rst/update_cyberok.sh | bash >> /var/log/cyberok.log 2>&1") | crontab -

Для MikroTik дома

Скачайте .rsc файл и импортируйте в роутер:

/import file=tspuips_mikrotik.rsc

Добавьте правило фаервола:

/ip firewall filter add chain=input protocol=tcp tcp-flags=rst src-address-list=TSPUIPS action=drop
